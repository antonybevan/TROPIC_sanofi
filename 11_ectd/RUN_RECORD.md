With the official DTDs in `util/dtd/`, **all three eCTD XML files pass `xmllint --noout --valid`**:

| File | DTD | Result |
|---|---|---|
| `index.xml` | `ich-ectd-3-2.dtd` (ICH eCTD 3.2) | DTD-VALID |
| `m1/us/us-regional.xml` | `us-regional-v3-3.dtd` (FDA US Regional **v3.3**) | DTD-VALID |
| `.../tropic/stf-tropic.xml` | `ich-stf-v2-2.dtd` (ICH STF v2.2) | DTD-VALID |

Real bugs the DTDs surfaced and fixed in `build_ectd_backbone.py`:
- **xlink #FIXED typo** — ICH eCTD 3.2 and FDA us-regional v3.3 both hard-code the misspelled
  `http://www.w3c.org/1999/xlink`; the generator now emits the value the DTDs require.
- **`indication` #REQUIRED** added to `m5-3-5-reports-of-efficacy-and-safety-studies`.
- **us-regional migrated v2.01 → v3.3** (FDA retired v2.01; v3.3 required since 2022-03-01): new
  namespace `http://www.ich.org/fda`, `applicant-info` subtree, and the required
  `application-containing-files` / `application-type` / `submission-type` / `submission-sub-type`
  attributes. All identifiers are labelled **EXAMPLE** (`000000`) — replace with FDA-assigned
  values before any real submission.
- **STF retargeted** to ICH STF v2.2 (`dtd-version` 2.3 → 2.2); structure already conformed.

DTD files are tracked; the download `.zip`/`.htm` and the FDA spec PDFs are git-ignored.
